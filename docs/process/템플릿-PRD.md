## 이 PRD를 프롬프트로 쓸 때 (순서가 중요)

실제로 LLM에 붙여넣을 때는 이 순서로 두면 결과가 좋아집니다.

1. **하네스 + 참조 패턴 + 출력 형식** (생성 지시 블록을 맨 위로)
2. 사용자와 목표
3. **데이터/콘텐츠 예시** (구체적일수록 좋음)
4. **필수 상태** (열거해서 "이 상태들을 모두 생성하라"고 지시)
5. **제약/안티패턴** (명시적 "하지 말 것")
6. 판단·수용 기준 (모델이 더 나은 선택을 하도록 맥락 제공)

PRD를 프롬프트로 튜닝할 때 손볼 곳은 대부분 [생성 지시] 필드입니다. [판단 맥락] 필드는 모델에 넣어도 무방하지만 주 목적은 리뷰용입니다.

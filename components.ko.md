# Synapse components

**이것은 닫힌 집합입니다.** 아래의 50개 컴포넌트 항목(각각 `##` 제목 하나이며, 일부 항목은 형제 컨트롤을 묶습니다. 예: Checkbox · Radio · Switch)은 생성되는 UI에서 허용되는 유일한 구성 요소입니다. 자주 쓰이는 다중 컴포넌트 조합은 `recipes.md`에 명세되어 있으니, 처음부터 조합하기 전에 그곳을 먼저 확인하십시오. 이 컴포넌트들이나 문서화된 조합(`patterns.md`)으로 충족할 수 없는 요구가 있다면, 올바른 조치는 `design.md` §6에 따라 에스컬레이션하는 것입니다. 변형을 발명하거나, 속성을 추가하거나, 기존 컴포넌트를 다시 스타일링하는 것은 결코 올바르지 않습니다.

모든 명세는 동일한 스키마를 따릅니다. `variants`와 `sizes`는 남김없는 열거입니다. `forbidden`은 에이전트가 가장 흔히 시도하지만 해서는 안 되는 수정 사항을 나열합니다. 컴포넌트는 단일 시스템 사이즈 스케일을 사용합니다; Table은 기본적으로 컴팩트하게 동작합니다(foundations §4).

관례: 높이는 `--sy-control-height-*`(`tokens/synapse.css`)를 가리킵니다. "포커스 링"은 `--sy-border-focus`의 밀착 2px 링(box-shadow라 모서리 반경을 감쌈) — 단, Button은 버튼 자신의 색을 ~50% `color-mix`로 밝게 한 틴트(v6.58.1).

## Button

**목적:** 액션을 트리거합니다. 내비게이션용이 아닙니다(Link를 사용하십시오). 버튼은 결코 URL을 변경하지 않습니다.

**변형(닫힘):**

| Variant | Style | Use |
|---|---|---|
| `primary` | `action.primary-bg` 채움, `action.primary-fg` 텍스트 | 한 영역의 유일한 주요 액션. 영역당 최대 하나만 보입니다. |
| `secondary` | 토널: `action.secondary-bg` 채움, 테두리 없음, `fg.primary` 텍스트(v5.0 — 아웃라인 스타일을 대체함. 호버 시 `secondary-bg-hover`로 단계 이동) | 표준 액션. 기본 선택지. |
| `ghost` | 투명, 테두리 없음, `fg.secondary` 텍스트 | 낮은 강조, 툴바, 반복되는 행 액션. 투명/토널의 구분이 ghost와 secondary를 구별 가능하게 유지합니다. |
| `danger` | `status.danger-bg-solid` 채움(밝은 쪽의 red.400, ~3.4:1 — §8 정책), semibold 600의 흰색 텍스트. 호버/눌림 시 `danger-bg-solid-hover`(red.500, AA)로 어두워짐 | 파괴적 확인 전용. 확인 대화상자 안이나 명시적 의도 이후에만. |
| `brand` | `action.brand-bg` 채움 | **포인트 / 브랜드 컬러 `#0621C4`(변형 accent→brand 개명, v6.58).** 관할: 브랜드 아이덴티티 요소 + 대화형 AI CTA — Ask agent / Composer send, 화면당 최대 1. 운영성 에이전트 액션(Run/Retry/Resume)은 `primary`/검정 유지 — 설정된 실행을 수행하는 것은 결코 포인트 컬러를 얻지 않으며, 오직 에이전트를 대화형으로 호출할 때만 얻습니다. 흰색 레이블은 AA를 충족합니다(흰색 위 `#0621C4` ≈ 10:1). |

**크기:** `sm`(height-sm, 라벨 13/12), `md`(height-md, 기본), `lg`(height-lg, focus 모드 히어로 전용).
**상태:** default, hover(`primary`: bg-hover; `secondary`: `secondary-bg-hover`; `ghost`: `bg.hover`), active/pressed(설계상 hover와 동일 — 눌림 피드백은 세 번째 채움이 아니라 클릭 자체에서 옵니다), focus-visible(링), disabled(`fg.disabled`, `bg.disabled` 채움; 40% 불투명도는 금지), loading(16px 스피너가 선행 아이콘을 대체함; 라벨은 유지; 너비는 변경되어서는 안 됨).
**Pill 옵션(v5.0):** Guided 아키타입 히어로와 empty-state 최초 사용 순간의 `primary` 버튼은 `radius.full`을 사용할 수 있습니다. 폼, 툴바, 표, 밀집 영역에서는 절대 안 됩니다. pill 실루엣은 히어로 표식이지 일반 스타일이 아닙니다.
**해부:** 선택적 선행 아이콘(16px) + 라벨. **광학적 패딩 트림(v6.17.2):** 아이콘+텍스트 버튼에서 아이콘 쪽 패딩은 `--sy-control-padding-x`에서 2px를 뺀 값입니다(16px 스트로크 아이콘은 내부 여백을 지니므로, 동일한 패딩은 아이콘 쪽이 더 무겁게 읽힘). 이는 쪽마다 적용됩니다. 선행 아이콘은 왼쪽을 트림하고, 후행 셰브런/외부 표식은 오른쪽을 트림합니다. 아이콘 전용 버튼은 예외입니다(정사각형이므로). 아이콘 전용 버튼은 다음에만 허용됩니다: 닫기, 더 보기(⋯), 편집, 삭제, 복사, 새로 고침, 펼치기/접기, 설정 — 그리고 반드시 `aria-label`을 가져야 합니다. **아이콘 전용 버튼은 항상 정사각형입니다**(v6.2.4): 너비는 해당 크기의 컨트롤 높이와 같습니다(sm 32→32, md 36→36). 정사각형이 아닌 아이콘 버튼은 변형이 아니라 버그입니다.

**라벨 및 아이콘 규칙(v5.0.3 — 닫힌 정책):**

- **텍스트 전용이 기본입니다.** 버튼은 아이콘을 획득하는 것이지, 장식을 위해 얻는 것이 아닙니다.
- **아이콘 + 텍스트는 다음의 경우에만 허용됩니다:** (a) 대화형 AI 진입 버튼일 때(v6.19부터 `primary`; 에이전트 글리프가 그 표식), 또는 (b) 버튼이 툴바/필터바 맥락에 있고 그 아이콘이 승인된 아이콘 액션 목록에 속하여, 반복되는 컨트롤 전반의 스캔을 아이콘이 돕는 경우. 그 외 어디서든 — 대화상자 푸터, 폼, 페이지 헤더 — 텍스트 전용입니다.
- **후행 아이콘:** 셰브런(메뉴를 여는 버튼, 분할 버튼)과 외부 링크 표식만. 후행 장식 아이콘은 절대 안 되며, 한 버튼에 아이콘 두 개도 안 됩니다.
- **라벨 텍스트:** `label` 타이포그래피 스타일(`danger`에서는 semibold); 문장형 대소문자; EN 동사 우선("Save changes") / KO 명사 또는 -하기 형태("변경사항 저장") — `content.md` §3.2에 따름; 종결 구두점 없음; "…"는 진행 중 동사("Saving…" / "저장 중…")이거나 실행 전 후속 단계를 여는 경우("Export…"는 형식 대화상자를 엶)에만.

**금지:** 고정 너비(KO/EN); 커스텀 색상; 영역당 하나를 초과하는 `primary`; 승인된 목록 밖의 아이콘 전용; ALL-CAPS 라벨; `sm` 외의 반경.
**접근성:** `<button>` 요소; Enter/Space로 활성화; loading은 `aria-busy`를 설정.
**이중 언어:** 라벨 크기는 콘텐츠와 `--sy-control-padding-x`에서 나옴; EN은 동사 우선("Save changes"), KO는 자연스러운 어순(변수 허용: "변경사항 저장"); 절대 잘라내지 않음.

## Link

**목적:** 내비게이션. `fg.link` 색상, 평상시 밑줄 없음, 호버/포커스 시 밑줄. 본문 텍스트 안의 인라인 링크는 항상 밑줄. 외부 링크는 16px 외부 아이콘을 덧붙임. **금지:** 링크를 버튼처럼 또는 그 반대로 스타일링; "click here" 라벨(KO: "여기를 클릭" 금지).

## Input (text)

**목적:** 한 줄 텍스트 입력. text, email, password, number, search를 포함합니다(search는 선행 16px 아이콘 + 채워졌을 때 지우기 버튼을 추가).

**해부(v5.0 — 채움형):** 라벨(필수, `--sy-label-size` medium, 위) · 필드(height-md, `bg.sunken` 채움, 테두리 없음, 반경 `sm`) · 선택적 헬퍼 텍스트(caption, `fg.secondary`) · 오류 텍스트(caption, `status.danger`, 헬퍼를 대체).
**상태:** default(채움형, 테두리 없음), hover(채움이 `bg.selected-hover`로 단계 이동), focus(1px `border.focus-input` 둘레 — 중립, v6.10 — + 채움이 `bg.page`로 전환: 입력을 위해 필드가 "열림". 클릭-투-타이프는 포커스를 항상 노출하고 파란색이 강조를 과다 노출했기 때문에, 입력 표면은 중립 키 톤으로 포커스를 나타냄; 비입력 컨트롤은 밀착 포커스 링을 유지(Button은 자신의 색, v6.58; 그 외 `border.focus`). 채움형 필드에는 포커스 링 없음(v6.8.1) — 테두리 교체 + 채움 변화가 복합 지표를 형성. 오류 필드는 포커스 상태에서 `border.error`를 유지; 채움 변화가 포커스를 전달), disabled(`bg.disabled`, `fg.disabled`), error(채움형 필드에 1px `border.error` + 오류 텍스트; 오류 텍스트는 단지 "invalid"가 아니라 반드시 수정 방법을 명시), read-only(`bg.surface`, 호버 반응 없음). Select, Combobox, DatePicker 트리거는 이 채움형 해부를 상속합니다.
**크기:** `md`만.
**부착물(v6.0):** 선택적 선행 레지스트리 아이콘(16px, `fg.tertiary` — search의 돋보기는 이 일반 슬롯의 한 인스턴스)과/또는 후행 부착물: 단위/형식 접미사(`fg.tertiary`, 예: "KRW", "%") 또는 레지스트리 아이콘 하나(예: 비밀번호의 eye/eye-off 표시). 선행 하나 + 후행 하나 최대; 부착물은 채움 안에 앉고 포커스를 받지 않음.

**금지:** 라벨로서의 플레이스홀더(플레이스홀더는 예시 콘텐츠일 뿐, `fg.tertiary`); 플로팅 라벨(한글 메트릭에서 깨짐); 번역 대상 콘텐츠에 240px 미만의 고정 필드 너비; 표 인라인 편집 셀을 제외하고 라벨을 시각적으로 숨기기.
**접근성:** 항상 `<label for>`; 오류는 `aria-describedby`로 연결; 오류 시 `aria-invalid`.
**이중 언어:** 라벨은 필드 위에(옆 좌측 정렬 금지 — KO/EN 라벨 너비 차이가 정렬을 깨뜨림); 헬퍼/오류 텍스트는 줄바꿈, 절대 잘라내지 않음.

## Textarea

여러 줄 Input. 최소 높이 3행, 세로 리사이즈만, 그 외에는 모든 Input 규칙을 상속합니다. 제한이 있을 때 문자 카운터(caption, `fg.tertiary`, 우측 하단) — 바이트가 아니라 문자를 셉니다(한글). **`autogrow` 변형(v6.0):** 콘텐츠에 따라 1행부터 선언된 최대치(기본 8)까지 커진 뒤 내부적으로 스크롤; 리사이즈 핸들 제거 — Composer의 동작이며, 이제 인라인 폼(댓글, 설명)에서도 사용 가능합니다.

## Select

**목적:** 5–15개의 알려진 정적 옵션에서 하나를 선택. 5개 미만: Radio 사용. 15개 초과, 비동기, 또는 사용자가 아는 값: Combobox 사용.
트리거는 후행 셰브런(16px)이 있는 Input 해부로 렌더됩니다. 메뉴는 Popover입니다: `bg.raised`, `border.default`, 반경 `md`, `shadow.overlay`; 옵션 height-md, 호버 시 `bg.hover`, 선택 시 `bg.selected` + 선행 16px 체크.
**상태:** Input과 같고, open 추가(셰브런이 180° 회전, `fast` 지속).
**금지:** 네이티브 `<select>` 스타일링을 커스텀 메뉴와 혼합; 칩 렌더링 없는 다중 선택(트리거 안에 input Chips 사용); 480px보다 넓거나 트리거보다 좁은 메뉴.
**접근성:** listbox 패턴; 전체 키보드(방향키, Home/End, 타입어헤드); Esc로 닫힘.
**이중 언어:** 메뉴 너비는 활성 로케일의 가장 긴 옵션에 맞춤; 고정 트리거 너비 없음.

## Checkbox · Radio · Switch

| Control | Semantic | Never |
|---|---|---|
| Checkbox(16px 박스, 반경 `xs`) | 폼에서 독립적인 on/off; 목록에서 다중 선택 | 즉시 효과 토글로서 |
| Radio(16px 원) | 2–5개의 보이는 옵션 중 정확히 하나 | 5개 초과 옵션에(Select 사용) |
| Switch(32×18 트랙) | 즉시 효과 토글, 즉시 적용 | 저장 버튼이 있는 폼 안에서 |

체크 상태: `action.primary-bg` 채움(검정/흰색 — 키 색상, 파랑 아님). 라벨은 항상 오른쪽, 클릭 가능, `--sy-body-size`.

**혼합 상태**(이질적인 값을 나타냄, 예: 대량 선택 부모나 다중 객체 편집):

| Control | Mixed rendering | Interaction |
|---|---|---|
| Checkbox | 키 색상 채움 + 수평 마이너스 바(`aria-checked="mixed"`) | 클릭 → 모두 체크; 두 번째 클릭 → 모두 체크 해제 |
| Switch | 안에 마이너스 글리프가 있는 중앙 정렬 썸(`fg.secondary`, 6×2px), 트랙 `border.strong` 채움(`aria-checked="mixed"`) — 트랙은 결코 키 색상을 취하지 않음: 채워진 트랙은 즉시 효과 컨트롤에서 순간적으로 ON을 잘못 신호함 | 토글 → 모두에 ON 적용(파괴적 OFF는 결코 기본값이 아님) |
| Radio | 혼합 상태가 존재하지 않음 — 이질적인 라디오 그룹은 **선택 없음** + 캡션 "Mixed values" / "여러 값"으로 렌더 | 선택하면 모두에 적용 |

**혼합 값 관례(컴포넌트 간).** 하나의 컨트롤이 현재 값이 서로 다른 여러 객체를 편집할 때는 언제나 컨트롤이 혼합 표식을 보여줍니다 — 한 객체의 값을 공유된 것처럼 보여주지 않습니다. 텍스트형 컨트롤(Input, Select, Combobox, DatePicker)은 `fg.tertiary`로 플레이스홀더 스타일의 문자열 "Mixed" / "여러 값"을 표시하며; 어떤 값을 입력하든 모두 덮어씁니다. 지워진 혼합 필드는 빈 상태가 아니라 "Mixed"로 돌아갑니다.
**접근성:** 내부적으로 네이티브 input; 라디오 그룹은 `fieldset/legend` 사용; 혼합 상태는 `aria-checked="mixed"` 사용.

## Badge

**목적:** 컴팩트한 정적 주석 — 상태, 카운트, 카테고리. 결코 상호작용하지 않음(상호작용 = Chip). **기본 렌더링(v6.24):** 선언된 색상이 없는 Badge는 `neutral`로 렌더됩니다 — 배지는 결코 채움 없는 떠 있는 텍스트로 나타나지 않습니다.

**크기:**

| Size | Spec | Jurisdiction |
|---|---|---|
| `md`(기본) | 높이 20px, `micro` 텍스트, padding-x 8 | 어디서나: 표, 카드, 목록, 메타 행. 항상적인 인식 크기. |
| `lg` | 높이 24px, `label-sm` 텍스트(`solid`일 때 600 — foundations §8 무게 보정), padding-x 12 | `heading-xl`+ 제목(페이지 헤더, R1)과 히어로/empty-state 맥락 옆에서만. 표, 목록, 밀집 영역 안에서는 절대 안 됨 — 그곳에서의 크기 항상성이 상태를 스캔 가능하게 만듦. |

한 뷰에 한 크기, 형태와 마찬가지로.

**광학적 중앙 정렬:** 배지 라벨은 한 줄이며 고정 높이 안에서 flex 중앙 정렬됨; 줄 높이는 1로 접힘(컨테이너가 박스를 제공함 — 줄 높이 하한 규칙은 줄바꿈되는 텍스트를 겨냥하지, 봉인된 한 줄 컨테이너가 아님) 더하기 폰트가 예약하지만 짧은 라벨은 거의 쓰지 않는 디센더 공간을 보정하기 위한 1px 위쪽 패딩 미세 조정. 이것이 없으면 라벨이 pill 상단으로 치우쳐 읽힙니다. **스케일 종속:** 이 미세 조정은 `md` 배지의 촘촘한 20px/11px 지오메트리에만 적용됩니다. 24px/12px 비율(`lg` 배지, Chip)에서는 컨테이너에 광학적 여유가 충분하여 단순 flex 중앙 정렬이 옳고 미세 조정은 아래로 과보정합니다 — 두 표면 모두 경험적으로 검증됨.

**형태 변형**(뷰 수준의 스타일 선택, 인스턴스별로는 결코 아님 — 뷰당 한 형태):

| Shape | Radius | Use |
|---|---|---|
| `pill`(기본) | `full` | 어디서나 기본. |
| `rounded` | `sm` | 뷰가 선택한 형태로서 어떤 크기/맥락에서든 사용 가능. 밀집 표와 코드 인접 맥락에서는 *기본 기대치*이며, 그곳에서는 pill을 타일링하면 노이즈로 읽힙니다. |

**색상 변형:** `neutral`, `info`, `success`, `warning`, `danger`, `ai`(`ai.*` 토큰 사용 — 에이전트 관련 상태만), `category`(정적 분류 라벨: 20% 채움의 결정적 `viz-n` + 일치하는 텍스트, Avatar와 동일한 해싱; 시스템 할당, 사용자가 선택하지 않음). `category`는 `subtle` 강조만 지원합니다 — 분류는 결코 긴급하거나, 아웃라인이거나, 점으로 축소되지 않습니다.
**강조 변형**(색상과 직교; 주변 컨텍스트와 중요도로 선택):

| Emphasis | Rendering | Use |
|---|---|---|
| `subtle` (기본값) | `status.*-bg` 채우기, `status.*` 텍스트, 보더 없음 | 표, 카드, 헤더의 표준 상태입니다. |
| `solid` | 아래 색상별 규칙 참조 | 화면 건너편에서 스캔 가능해야 하는 고가시성 표식. 뷰당 solid 색상 최대 하나. |
| `outline` | 투명, 1px `status.*` 테두리, `status.*` 텍스트 | 은은한 채움이 화면을 색으로 타일링할 밀집 표. |
| `dot` | 8px `status.*-bg-solid` 점(중간 값 — 점은 비텍스트, 3:1 하한; 더 어두운 `status.*` 텍스트 토큰은 채움에 쓰이지 않음) + `body-sm` `fg.secondary` 텍스트, 컨테이너 없음 | 가장 조용한 선택지: 긴 목록, 사이드바 항목, 범례 같은 행. |

**색상별 solid 규칙.** solid는 스타일 선택이 아닙니다 — 모든 solid 배지에는 지정된 임무가 있습니다:

| Solid color | Rendering | The one job | Never |
|---|---|---|---|
| `info` / `success` / `warning` / `danger` | `status.*-bg-solid` + `fg.on-solid`(흰색, 모든 색조) | 두 가지 관할: (a) 화면 건너편에서 스캔 가능해야 하는 긴급 상태 — 실시간 인시던트, 차단성 실패 — 뷰당 solid 색상 하나로 제한; (b) 운영/모니터링 상태 열의 옵트인(그곳에는 뷰당 상한이 적용되지 않음); 표에서의 조용한 기본은 `dot`(v5.0). 채움은 흰색과 함께 ~3.5:1로 실행됨 — 문서화된 정책 이탈(foundations §8): 짧은 semibold 라벨만, 결코 문장은 아님. | 표 밖의 일상적 상태(그건 `subtle`); 두 단어 라벨보다 긴 것 뒤의 solid 채움. |
| `neutral` | `bg.inverse` + `fg.inverse` | **릴리스 표식 전용**: "New" / "신규", "Beta" / "베타", "Early access". 뷰당 최대 하나. 표식은 만료됨 — 한 릴리스 주기보다 오래된 "New"는 버그이며, 표식이 배포될 때 만료 소유권이 지정되어야 함. | 어떤 종류의 상태든; 일반적 강조; 카운트. 키 색상으로 렌더되므로, 오용은 주요 액션처럼 읽힘. |
| `ai` | `ai.solid` + `fg.on-solid`(v6.19, accent-bg였음) | **실시간 활동 비콘 전용**: 에이전트가 현재 표면에서 능동적으로 작동하는 동안 보임(자율 모드 실행 중, 대량 변경 진행 중 — 펄스가 있는 "실행 중" 허용). 활동이 멈추는 순간 반드시 사라짐. 화면당 최대 하나. | 출처나 에이전트 관련 정적 상태(그건 `ai` subtle); 지속적인 것; 같은 클러스터에서 `brand` Button에 인접한 배치 — solid 파랑은 "AI 작동 중"을 뜻하며, 액션 버튼 옆의 정적 배지는 둘 다 흐림. |

**카운트 배지:** 내비 항목과 탭의 숫자 전용 배지(`neutral` subtle, tabular-nums); 100 이상은 "99+"로 렌더.
**카운트 오버레이(v6.2.4–5)** — 아이콘 버튼 위에 앉는 카운트(NotificationCenter 벨이 그 유일한 관할): 18px 최소 너비/높이 미니 pill, `micro` 숫자, **`bg.inverse-soft` 채움 + `fg.inverse` 텍스트**(부드러워진 키 — 순수 검정의 거칢이나 상태 의미 없이 어떤 크롬에도 잘 보임), 하부 표면 색상의 2px 링, 버튼 우상단 모서리에 절반쯤 걸침(고정 top/right −2px)으로 pill이 글리프 우상단 일부를 덮음 — 링이 아래에 덮인 아이콘의 가독성을 유지함. 고정 오프셋, 결코 백분율 이동이 아니므로 모든 버튼 크기에서 덮임이 동일하게 읽힘. 여기서 neutral-subtle 채움은 금지 — 오버레이는 보이기 위해 존재함. 0에서 사라짐; 결코 "0"을 렌더하지 않음.
**`with-icon` 옵션(v6.0):** subtle 및 solid 상태 배지는 라벨 앞에 일치하는 12px 레지스트리 상태 아이콘을 지닐 수 있음 — 상태가 중요한 뷰에서 색맹 안전 스캔을 위한 아이콘 + 색상 + 텍스트의 삼중 중복성. 아이콘은 항상 상태 개념의 레지스트리 아이콘이며; 결코 장식적이거나, `dot`/`outline`(이미 최소) 또는 `category`에는 쓰지 않음.
**금지:** 상호작용하는 배지(Chip 사용); 문장 길이 콘텐츠; 색상 발명; 한 뷰 안에서 같은 상태 의미에 강조 수준 혼합; 한 뷰 안에서 형태 혼합; 위 표의 지정된 임무 밖의 solid.
**이중 언어:** KO 상태 용어는 흔히 2–4음절("진행 중", "완료") — 너비는 콘텐츠에서, 결코 고정 아님. 상태 어휘는 `content.md` §3.3의 닫힌 집합입니다.

## Chip

**목적:** 컴팩트한 **상호작용** 요소 — 제안을 선택, 정제, 제거, 또는 수락. 정적 대응물은 Badge이며; 구분은 절대적입니다: 클릭할 수 있으면 Chip이고; 정보만 전달하면 Badge입니다. 높이 24px, 반경 `sm`(rounded — pill 배지와 의도적으로 구별되어 상호작용성이 형태로 스캔됨), 텍스트 12, 단순 flex 중앙 정렬(Badge 광학 미세 조정은 여기 적용되지 않음 — 24px/12px에서는 컨테이너에 여유가 충분하여 미세 조정이 아래로 과보정함).

**변형(닫힘):**

| Variant | Rendering | Behavior & use |
|---|---|---|
| `input` | `bg.sunken`, `fg.primary`, 후행 12px ✕(`aria-label` 필수); 값이 사람이나 에이전트일 때 선행 Avatar 16을 지닐 수 있음(수신자/담당자 칩 — 형태 규칙 적용)(v6.0) | 제거 가능한 선택: Combobox 다중 값, 수신자 목록, 적용된 값. ✕는 제거; 칩 본체는 그 외에는 클릭 불가. |
| `filter` | 미선택: 투명 + 1px `border.default` + `fg.secondary` · 선택: `bg.selected` + 1px `border.strong` + `fg.primary` + 선행 12px 체크 | 필터바와 목록 헤더에서 토글 가능한 정제. |
| `category` | 20% 채움의 결정적 `viz-n` + 일치하는 텍스트(Avatar 해싱) | 클릭 가능한 분류 라벨 — 클릭하면 그것으로 필터. 같은 분류의 정적 표시는 Badge `category` 사용. |
| `suggestion` | **아웃라인** — `bg.raised` + `border.default` 헤어라인, `fg.primary`; 호버 `bg.hover` + `border.strong`(v6.38 — 소스 pill과 스타일 교체: suggestion은 액션이며 선명한 아웃라인이 클릭 가능하게 읽힘; 틴트는 강조-틴트 표식과 어울리는 출처로 옮겨감) | 에이전트가 제안한 다음 프롬프트/액션; 탭하면 삽입 또는 실행. Console과 empty state에서만 허용, 표면당 최대 3개. |

**상태:** default, hover(`bg.hover` 또는 `border.strong`), selected(filter/category), 포커스 링, disabled(`fg.disabled`, 상호작용 없음).
**금지:** 커맨드 버튼으로서의 칩(커맨드는 Button — 칩은 결코 주요 또는 파괴적 액션을 수행하지 않음); 한 행에 `input`과 `filter` 혼합; 수동 색상 선택; 허용된 표면 밖의 `suggestion` 칩; 칩 라벨 잘라내기(대신 행을 줄바꿈).
**이중 언어:** 너비는 콘텐츠에서; 제거-✕ `aria-label`은 로케일화됨("Remove" / "제거").

## Card

**목적:** 관련 콘텐츠의 경계 지어진 그룹. 반경 `md`, 패딩 `--sy-card-padding`. 선택적 헤더(`heading-lg` 또는 `heading-md` + 선택적 액션)와 푸터를 full-bleed `border.subtle`로 구분.

**변형:**

| Variant | Rendering | Use |
|---|---|---|
| `flat`(기본, v5.0) | `bg.surface`, 테두리 없음 | 표준 카드: 박스가 아니라 표면 단계와 간격으로 그룹화. 또한 허용된 내부 그룹화(테두리가 있는 카드-인-카드 중첩은 여전히 금지). |
| `outlined` | `bg.raised`, 1px `border.default` | 진정으로 분리 가능한 객체: 선택 가능한 그리드의 항목, 임베드된 데이터 영역, `interactive`한 모든 것(클릭 가능성에는 가장자리가 필요함 — `interactive`는 `outlined` 또는 `elevated`를 함의). |
| `elevated` | `bg.raised`, 1px `border.default`, `shadow.raised` | Focus 아키타입의 핵심 순간만(페이지가 다루는 그 하나); 페이지당 최대 하나. 유일하게 허용된 정적 카드 그림자. |
| `ai` | `ai.surface`, 1px `ai.border` | 에이전트가 생산한 콘텐츠를 위한 컨테이너. ProposalCard는 이 변형의 특수화. |
| `stat` | 고정 해부를 가진 `outlined` 카드: `label` + `fg.secondary` 제목 · `stat`/`stat-lg` 값 · 선택적 델타 행 · 선택적 스파크라인. (v6.11 `emphasized` 옵트인은 v6.40에서 제거됨 — 유지관리자: stat 카드에 슬레이트 없음; 메트릭 그리드는 균일하게 렌더, 긴급성은 큐와 상태에 속하지 카드 틴트가 아님) | KPI 표시. 그리드 프리셋은 `recipes.md` 참조. |

**수정자**(어떤 변형과도 결합): `interactive`(카드 전체 클릭 가능: 호버 `border.strong` + `bg.hover` + foundations §6에 따른 선택적 `hover-lift`; 정확히 하나의 액션; 키보드 포커스 가능), `selected`(1px `border.selected` 아웃라인 — 포커스가 아니라 선택).
**금지:** `elevated` 밖의 그림자; 테두리 있는 카드 중첩(`flat` 사용); 페이지 레이아웃 골격으로서의 카드; 비에이전트 콘텐츠에 `ai` 변형.

## Table

**목적:** 밀집 모드의 일꾼. 정렬된 열을 가진 구조화된 레코드.

**선택 열(v6.24.1):** 40px 고정, 셀 패딩 0, 양 축 중앙 콘텐츠(체크박스에 `vertical-align: middle`) — 선택 셀은 텍스트가 아니라 컨트롤을 담으므로 텍스트 셀 패딩/정렬을 상속하지 않음. 헤더 체크박스 = Checkbox 규칙에 따른 혼합 상태를 가진 전체 선택.

**해부(강조 옵트인, v6.11):** 한 열 또는 셀 범위가 `emphasis.surface` 채움(+ 헤더 라벨에 `emphasis.fg`)을 취하여 현재 기간, 합계, 또는 비교 대상을 표시할 수 있음 — 표당 강조 열 최대 하나, 같은 셀에서 행 선택 틴트와 결코 결합하지 않음. 헤더 행(`--sy-label-size` medium, `fg.tertiary`, **채움 없음** — v6.5: 헤더 배경은 테두리 있는 표와 없는 표 모두에서 투명; 헤어라인 하단 규칙만이 헤더를 표시하고, 정렬 글리프는 예약 공간을 얻어 정렬 시 열이 이동하지 않음, sticky) · 행(높이 `--sy-table-row`, `border.subtle` 구분선) · 선택적 푸터/페이지네이션. **프레이밍(v4.0):** focus 표는 프레임 없음 — 페이지 배경 위 맨 헤더에 아래 단일 헤어라인 규칙만, 외곽 테두리 없음, 헤더 채움 없음; 밀집 및 수평 스크롤 표는 프레임 유지(1px `border.default`, 반경 `md`, 클립됨, `bg.surface` 헤더) — 고정 열과 스크롤 가장자리에 경계가 필요하기 때문.
**열 규칙:** 텍스트는 좌측 정렬; 숫자는 `font-variant-numeric: tabular-nums`로 우측 정렬(ID는 mono); 날짜/시간은 표당 하나의 일관된 형식; 상태는 Badge로; 행 액션은 행 끝의 ghost 아이콘 버튼으로, 밀집 모드에서는 호버 시 보이고 focus 모드에서는 항상 보임.
**동작:** 호버 `bg.hover`; 선택 `bg.selected` + 선행 체크박스; 정렬 가능한 헤더는 후행 12px 화살표를 얻음(활성 정렬 열만, 한 번에 하나). Empty state: 프레임 안의 EmptyState 컴포넌트, 결코 맨 "no data" 문자열이 아님. Loading: 열 레이아웃에 맞는 Skeleton 행.
**셀별 상태:** 툴팁이 있는 잘라내기 허용(목록 행 외에 유일하게 허용된 잘라내기 지점).
**금지:** 얼룩말 줄무늬(구분선으로 충분); 표 안 강조 색상 1개 초과; 고정된 첫 열 없는 수평 스크롤; 표를 가장한 행별 카드 레이아웃.
**이중 언어:** 열 최소 너비는 더 넓은 로케일에 맞춤; 헤더 잘라내기 금지.

**고급 동작(v1.3)** — 모든 Table에서 사용 가능; 각각 뷰별 옵트인:

- **열 컨트롤.** 헤더 가장자리를 드래그하여 리사이즈(최소 80px; 사용자별 뷰별 지속). 왼쪽으로만 고정, 최대 2열; 고정된 열은 `border.strong` 오른쪽 가장자리를 보이고 스크롤 중 `bg.surface`/행 배경을 유지. 헤더 열 메뉴(⋯)를 통해 숨기기/보이기 및 재정렬. 열 메뉴 항목 집합은 닫힘: Sort ascending / Sort descending / Pin / Unpin / Hide / Resize to fit.
- **대량 선택.** 선행 체크박스 열(헤더 체크박스 = 페이지 전체, 부분 선택 시 미결정). 1개 이상 행이 선택되면 표 툴바가 선택 바로 대체됨: `bg.selected` 스트립, "14 selected" / "14개 선택됨"(13 medium), 액션 버튼 최대 4개(`ghost`) + 오버플로 메뉴 + Clear. 파괴적 대량 액션은 카운트와 함께 Modal로 확인.
- **인라인 편집.** 텍스트, 숫자, select 셀에만 허용. 더블 클릭 또는 포커스된 셀에서 Enter로 편집 진입; 편집기는 표준 포커스 링을 가진 셀 안 테두리 없는 형태(이것이 허용된 숨겨진 라벨 예외). Enter는 커밋, Esc는 취소, Tab은 커밋 + 이동. 잘못된 값: `status.danger` 셀 테두리 + 수정을 명시하는 Tooltip; 셀은 유효해지거나 취소될 때까지 편집을 벗어나지 않음.
- **행 그룹화.** 한 수준만. 그룹 헤더 행: 32px, `bg.surface`, 13 medium 라벨 + 카운트 Badge, 셰브런으로 접기 가능. 집계(합계/카운트)는 그룹 헤더에 우측 정렬, tabular-nums로 렌더.
- **가상화.** 200행 초과 시 필수. 이 때문에 행 높이는 고정됨 — 가변 높이 행은 가상화된 표에서 금지.
- **펼침 행.** 선행 셰브런 열; 펼치면 행 아래에 상세 패널(`bg.surface`, 전체 행 너비, 자체 패딩)이 드러남. 한 수준; 펼쳐진 패널은 `flat` Card나 DescriptionList를 담을 수 있으나 결코 다른 표는 아님.
- **합계/요약 행.** 하단 고정 행, `bg.surface`, `label` 타입, 값은 tabular-nums; 셀에 집계를 명시("Σ 1,204" 또는 "avg 4m 02s") — 결코 라벨 없는 숫자가 아님.
- **헤더 추가물.** 열 헤더는 정보 아이콘(13px, 열 정의를 담은 Tooltip을 엶)과 `fg.tertiary`의 단위 접미사를 지닐 수 있음("Duration *(min)*" — 단위는 헤더에, 셀마다 반복하지 않음).

**셀 렌더러(닫힌 집합).** 모든 열은 정확히 하나의 렌더러를 선언함; 렌더러가 정렬, 형식, 잘라내기 동작을 고정함. 이 집합은 새로운 데이터 형태가 결코 셀 UI 발명을 요구하지 않도록 존재함:

## Combobox

**목적:** 크거나(15개 초과), 비동기 로드되거나, 사용자가 아는 옵션 집합에서 하나 또는 여럿을 선택. 이것은 Select의 큰형제 — Select는 5–15개 정적 옵션에 머무름.

**해부:** Input 스타일 트리거(라벨 위, height-md, 후행 셰브런). 입력하면 즉시 필터링; 일치하는 부분 문자열은 하이라이트됨(600 무게, 색상 변화 없음). 메뉴 = Popover 표면, 옵션은 Select와 같음; 비동기 결과는 로딩 중 Skeleton 행 3개를 보임.
**다중 선택:** 선택된 값은 트리거 안에 제거 가능한 input Chips로 렌더되어 최대 2행까지 줄바꿈되고, 그 후 "+N" 오버플로 Chip(클릭 → 전체를 나열하는 popover). 메뉴 옵션은 선행 Checkbox를 얻음; 메뉴는 선택 사이에 열린 채로 유지; 접혔을 때 트리거 플레이스홀더는 선택 카운트가 됨.
**상태:** Input과 같고, open, loading, no-results 추가("No matches for '{query}'" 행 + 선택적 "Create '{query}'" 액션 — 생성은 데이터 모델이 명시적으로 허용하는 곳에서만, 플러스 아이콘이 있는 메뉴 항목으로 스타일링, 결코 blur 시 자동 생성 아님).

**편의 기능(모두 인스턴스별 옵트인):**

- `search-in-menu` — 트리거가 input이 아니라 버튼일 때(예: Modal이나 툴바 안), 메뉴가 상단에 테두리 없는 검색 행(full-bleed 하단 규칙)과 함께 열리며, 열릴 때 포커스됨.
- `select-all` — 다중 선택 메뉴는 혼합 상태 체크박스 동작을 가진 고정 "Select all ({n})" / "전체 선택 ({n}건)" 행으로 시작할 수 있고, 같은 행의 후행 위치에 "Clear" 액션을 더함. *필터링된* 집합에 작동하며, 필터가 활성일 때 그렇다고 말함("Select all 12 matching").
- `grouped` — sticky 그룹 라벨(`micro`, `fg.tertiary`) 아래의 옵션; 그룹은 50개 초과 옵션의 메뉴에서만 접힘.
- `descriptions` — 옵션 행은 `caption` `fg.secondary` 둘째 줄과/또는 선행 16px 아이콘을 지닐 수 있음; 행 높이가 40px로 커짐.
- `recent` — 사용자 자신의 선택에 기반한, 모든 그룹 위에 고정된 최대 3개 항목의 "Recent" 그룹.
- `async-more` — 페이지 매김 소스는 "Load more" 행(`ghost` 스타일링)을 덧붙임; 결코 메뉴 안 무한 스크롤이 아님.
- `virtualized` — 100개 초과 옵션에서 필수.

**접근성:** ARIA combobox 패턴; 다중에서 마지막 Chip을 제거하는 Backspace를 포함한 전체 키보드.
**금지:** 5개 미만 옵션(Radio)이나 5–15개 정적(Select)에 Combobox; 명시적 생성 가능 모드 없는 자유 텍스트 값; 트리거보다 좁은 메뉴; 선택된 Chip 잘라내기(대신 줄바꿈); 파괴적 대량 변경을 기본으로 하는 select-all.
**이중 언어:** 필터링은 두 로케일의 라벨과 로마자화된 한글에 일치; 옵션과 Chip 너비는 콘텐츠에서.

## DatePicker

**목적:** 날짜, 날짜 범위, 시간 선택. 형식화된 Input 트리거 + 캘린더 Popover.

**변형(닫힘):** `date`(기본) · `range` · `datetime` · `time`.

**캘린더 해부:** 헤더(월/년 라벨 14 semibold + 이전/다음 아이콘 버튼) · 요일 행(11 medium, `fg.tertiary`) · 7×6 날짜 그리드. 날짜 셀 32px, 반경 `sm`: 오늘 = `emphasis.surface` 채움 + `emphasis.border` 인셋 헤어라인 + `emphasis.fg` semibold 숫자(v6.11 — now 표식 임무); 선택 = `action.primary-bg` 채움; 범위 내부 = 각진 가장자리의 `bg.selected`, 끝점 채움; 다른 달 날짜 `fg.disabled`; 비활성 날짜 `fg.disabled` + 취소선 금지 — 대신 호버 없음 + `aria-disabled` 사용.
**`range`:** 나란히 놓인 두 캘린더(밀집/좁은 맥락에서는 하나); 왼쪽에 프리셋 레일 — 닫힌 프리셋 집합: Today, Last 7 days, Last 30 days, This month, Last month, Custom("오늘", "지난 7일", "지난 30일", "이번 달", "지난달", "직접 선택"). 하부 값이 datetime 창(예약 창, 로그 쿼리)일 때 각 끝점은 `datetime` 규칙에 따라 시간 필드를 지닐 수 있음; 그렇지 않으면 범위는 날짜 전용.
**`datetime`:** 캘린더 Popover가 full-bleed `border.subtle` 규칙 아래에 시간 행 푸터를 얻음: 16px 시계 아이콘 + 시간 필드 + 시간대 라벨(`caption`, `fg.tertiary` — 필수, 결코 모호하지 않음). 시간 입력은 타이핑, 24시간 `HH:MM`(content.md §6); 방향키는 포커스된 세그먼트에서 15분씩 이동; 타이핑된 값은 blur 시 정규화("930" → 09:30). 트리거는 결합된 로케일 형식을 표시("2026년 1월 9일 14:02 KST" / "Jan 9, 2026, 14:02 KST").
**`time`:** 독립형 시간 필드(캘린더 없음) — 동일한 입력 규칙, 너비는 콘텐츠에 맞춤. 기간에는 시간 선택기가 아니라 Input `number` + 단위를 사용.
**형식:** 로케일별 표시 — EN `Jan 9, 2026`, KO `2026년 1월 9일`; 타이핑 입력은 로케일의 숫자 형식(`01/09/2026` / `2026-01-09`)을 받아들이고 blur 시 정규화. 주 시작은 로케일 관례를 따름(ko-KR과 en-US 모두: 일요일; 명시적 워크스페이스 재정의를 존중). 시간대가 중요할 때(예약, 로그)는 값 옆에 tz 라벨을 표시 — 결코 모호하게 두지 않음.
**상태:** Input과 같고, open, invalid-date 오류(받아들여지는 형식을 명시), min/max 경계(범위 밖 날짜는 Tooltip에 이유와 함께 비활성).
**금지:** 선택기 없는 텍스트 전용 날짜 입력; 단위별 드롭다운(일/월/년 Select) — 생년월일 스타일의 과거 입력 제외; 단일 날짜에 두 캘린더; 호버 시 절대값 없는 상대 전용 표시.

## SegmentedControl

**목적:** 즉각적 효과로 2–5개 동급 뷰나 파라미터 사이의 배타적 전환 — 차트 기간(1D/7D/30D), 레이아웃 토글(list/grid). Tabs(객체 측면)가 아니고, Radio(폼 데이터, 지연된 효과)도 아님.

**해부:** 컨테이너 `bg.sunken`, 반경 `sm`, **4px 내부 패딩**(v6.8 — 동심 모서리 규칙, foundations §5: 내부 반경 = 외부 − 인셋; 8 − 4 = 4가 두 반경을 스케일 위에 유지하고, 조립된 컨트롤이 정확히 컨트롤 높이(36)에 안착). 세그먼트: 높이 28, 반경 `xs`, padding-x 12, 13 medium, `fg.secondary`; 선택된 세그먼트 `bg.page` 채움 + 1px `border.default` + `fg.primary`. 콘텐츠 기반의 동등한 너비; 컨트롤은 콘텐츠에 맞춰 크기 조정.
**상태:** default, hover(`fg.primary`), selected, disabled(컨트롤 전체만 — 개별 세그먼트는 결코 아님), 활성 세그먼트의 focus-visible 링.
**접근성:** `radiogroup` 시맨틱; 방향키로 선택 이동.
**금지:** 세그먼트 5개 초과(Select 사용); 승인된 아이콘 목록 밖의 아이콘 전용 세그먼트; 한 컨트롤에 아이콘+텍스트와 텍스트 전용 세그먼트 혼합; 내비게이션이나 폼 제출에 사용.
**이중 언어:** 세그먼트 너비는 콘텐츠에서 — "지난 30일"과 "Last 30 days"가 둘 다 잘림 없이 맞아야 함.

## Accordion

**목적:** 2차 콘텐츠의 점진적 공개 — 고급 설정, FAQ 스타일 상세, 원시 페이로드.

**해부:** `border.subtle`로 구분된 항목. 헤더 행: 40px, 셰브런(16px, `fast`로 90°→ 아래로 회전), 14 medium 제목, 선택적 우측 정렬 메타(`fg.tertiary`, 13). 패널: 본문 텍스트, 패딩 0 0 16px, 제목 가장자리에 들여쓰기. 높이는 `base`로 애니메이션 — 허용된 높이 애니메이션 예외.
**동작:** 여러 항목이 동시에 열릴 수 있음(기본); 단계형 콘텐츠에는 단일 열림 모드 허용. 상태는 세션 내에서 지속.
**상태:** collapsed, expanded, disabled 항목(`fg.disabled`, 셰브런 회전 없음).
**접근성:** 헤더는 `aria-expanded`와 `aria-controls`를 가진 `<button>`.
**금지:** 접힌 항목 안에 주요 콘텐츠, 주요 액션, 또는 오류 상태 숨기기; 아코디언 중첩; Tabs나 표 그룹화의 대체물로서의 아코디언; 어포던스로서 셰브런 외의 아이콘.

## FileUpload

**목적:** 보이는 진행률과 복구 가능한 오류를 가진 파일 입력.

**변형:** `dropzone`(대시 1px `border.strong`, 반경 `lg`, 패딩 32×24, 중앙: 48px EmptyState 스타일 메달리온 안의 20px 업로드 아이콘(단일 −8px 헤어라인 링) + "Drop files here or **browse**"(13 medium, browse는 Link) + 제약 캡션 — "PDF, CSV up to 20MB" / "PDF, CSV · 최대 20MB")과 `button`(컴팩트한 맥락을 위한 `secondary` Button "Attach file" / "파일 첨부"). 드래그 오버 상태(v6.14): `border.focus-input` 테두리 + `emphasis.surface` 채움. 대시 테두리는 드롭 대상에만 계속 허용됨.
**파일 행:** 입력 아래 각 40px — 16px 파일 유형 아이콘, 파일명(전체 이름을 Tooltip에 담은 가운데 생략 잘라내기), 크기(`fg.tertiary`, tabular-nums), 그 후 상태별: 업로드 중 = 행 하단을 가로지르는 4px 확정 ProgressBar + 퍼센트; 성공 = `status.success` 체크; 오류 = `status.danger` 아이콘 + 한 줄 원인 + Retry ghost 버튼; 모든 행은 제거 ✕ 아이콘 버튼을 얻음(`aria-label` 필수).
**규칙:** 제약(유형, 크기, 개수)은 선택 전에 항상 보이고, 위반은 일반적 실패가 아니라 파일별로 명시됨. 여러 파일은 개별 진행률과 함께 병렬로 업로드; 결코 단일 결합 바가 아님.
**금지:** 보이는 진행률 없는 업로드; 크기 초과/잘못된 유형 파일의 조용한 거부; 폼에 다른 미입력 필수 필드가 있을 때 드롭 시 자동 제출.

## SplitPanel

**목적:** Workbench 아키타입의 리사이즈 가능한 인접 영역 — 목록 + 상세, 편집기 + 미리보기, 표 + 인스펙터. 컨테이너는 섹션 셸: 반경 `xl`, 1px `border.default`, 맞붙은 창(v6.9).

**해부:** 보이지 않는 8px 드래그 히트 영역을 가진 1px `border.default` 구분선으로 나뉜 창 2개(최대 3개). 호버/드래그 시 구분선: `border.strong`, 커서 `col-resize`. 구분선 중앙의 선택적 접기 셰브런(2차 창을 아무것도 없게 접음; 재열기 어포던스는 가장자리에 남음).
**동작:** 드래그는 최소 너비 안에서 리사이즈(콘텐츠 창 ≥ 280px, 레일/인스펙터 ≥ 200px); 구분선 더블 클릭은 기본 비율을 재설정; 비율은 사용자별 뷰별 지속. 창은 독립적으로 스크롤.
**금지:** 창 3개 초과; 수평 1 + 수직 1 수준을 넘는 중첩 분할; focus 아키타입의 SplitPanel(그 레이아웃은 고정); 최소 너비 없는 창(KO 라벨에 하한이 필요).
**접근성:** 구분선은 `aria-valuenow`를 가진 `role="separator"`, 포커스 시 방향키로 키보드 리사이즈 가능.

## Chart

**목적:** 표준화된 데이터 시각화. 차트는 16-semibold 헤더를 가진 Card 안에 위치; Card당 차트 하나.

**유형(닫힘):** `line`(추세; ≤8 시리즈), `area`(단일 시리즈만), `bar` 수직/수평(비교), `stacked-bar`(카테고리별 구성), `donut`(구성, ≤3 조각 — 그 외에는 bar), `sparkline`(인라인 24px, 축 없음, 단일 시리즈).
**해부:** 플롯 영역 · x/y 축(라벨 12 `fg.tertiary`, 축선 `border.default`) · 수평 격자선만(`border.subtle`) · 시리즈 1개 초과일 때만 범례(12px, 8px 정사각형 견본, 플롯 위 우측 정렬; 공간이 허락하면 직접 시리즈 라벨링을 선호) · 호버 툴팁(Tooltip 표면: 호버된 x값과 모든 시리즈 값을 보임, tabular-nums, 견본 키).
**색상:** 고정 순서의 `viz-1…8`; 단일 시리즈 = `viz-1`; 상태 인코딩 차트는 대신 `status.*` 토큰 사용. 결코 그라디언트, 결코 세 번째 차원으로서의 불투명도 램프.
**스케일 규칙:** 막대 차트는 y를 항상 0에서 시작. 선 차트는 보이는 축 단절 표식과 함께만 0 위에서 기준선을 잡을 수 있음. 숫자와 날짜 축 라벨은 로케일별 형식; 약어는 로케일 관례 사용(1.2k / 1.2천).
**상태:** loading = 스켈레톤 플롯(회색 막대/선 실루엣, 맥동); empty = 플롯 영역 안 EmptyState; error = Retry가 있는 오류 EmptyState. 데이터 업데이트는 스냅 — 새로 고침 시 전환 애니메이션 없음; 초기 그리기는 `base`로 한 번 애니메이션할 수 있음.
**크기:** 최소 높이 240px; sparkline 24px.
**금지:** 3D, 이중 y축, 3조각 초과 파이, 8 시리즈 초과(꼬리는 "Other" / "기타"로 집계), 플롯 안 장식 아이콘, y축 라벨 회전(대신 넓히거나 약어).

## Sidebar (app navigation)

**목적:** AgentOS의 단일 전역 내비게이션 표면.

## Modal

## Popover / Menu

앵커된 떠 있는 패널: `bg.raised`(모달 같은 L2 표면에서 열릴 때 `bg.raised-2`), `border.overlay`(v4.0), 반경 `md`, **6px 컨테이너 패딩**(v6.8 동심 모서리 규칙: 10 − 6 = 4 = 항목 반경 `xs`), `shadow.overlay`, `fast` 페이드+4px 이동 진입. 메뉴 항목: 높이 32px, 반경 `xs`, 13px, 선택적 선행 아이콘 16px, **항목 사이 4px 세로 간격**(v5.0.3–4 — 그것 없이는 인접한 호버와 선택 틴트가 융합; 2px는 지각 이하로 판명됨). 파괴적 항목은 `status.danger` 텍스트, 항상 마지막. 보이는 항목 최대 ~8개, 그 후 스크롤.
**구분선:** 1px `border.subtle`, 패널을 **가장자리에서 가장자리까지** 가로지름(컨테이너 패딩을 관통 — 규칙이나 테두리 있는 행을 패널 가장자리까지 음수 마진), 4px 세로 마진. 결코 인셋 아님. **이것은 모든 떠 있는 패널 내부의 모든 수평 규칙 AND 모달의 모든 패딩된 창 내부에 적용됨**(메뉴, popover, NotificationCenter, 후속 패널, 선택기 검색/푸터 행, 팔레트, Template Library의 열 내부 — v6.32) — 가장자리에 미치지 못하고 멈추는 규칙은 렌더링 버그로 읽힘(v6.21.2 재확인; 규칙은 v2.0로 거슬러 올라감). **full-bleed 구분선에 맞닿은 인셋 컨트롤(예: 푸터 버튼)은 네 면 모두 동일한 패딩을 취함 — 구분선과 컨트롤 사이 간격은 측면 및 바깥 가장자리 패딩과 MUST 같아야 함(v6.59.2); 비대칭 구분선 간격은 오정렬로 읽힘. (메뉴 탈출 행 같은 full-bleed 호버 행은 별개 패턴으로 행 패딩을 유지함.)**
**선택적 검색 행:** 8개 초과 항목의 메뉴는 테두리 없는 필터 입력(검색 아이콘, 13px, `border.subtle` 하단 규칙, full-bleed)으로 시작할 수 있음 — 일치 하이라이트를 포함하여 Combobox와 동일한 필터링 동작.
**금지:** popover 안 단일 컨트롤을 넘는 폼; 한 수준을 넘는 서브메뉴; 인셋 구분선.

## Tooltip

아이콘이나 잘린 문자열의 10단어 최대 명료화. `bg.raised-2`, 1px `border.default`, `fg.primary`, `caption` 타입, 반경 `xs`, 패딩 4/8, `shadow.overlay`, 300ms 호버/포커스 후 나타남. **Kbd 슬롯(v6.0):** 액션의 단축키를 후행 `.sy-kbd` 힌트로 덧붙일 수 있음("Copy ⌘C") — 제자리에서 단축키를 가르치는 허용된 방법. 동일 스킴 표면: 라이트 모드에서 라이트, 다크 모드에서 다크(v2.0에서 변경 — 반전 표면이 중립 필드에 비해 너무 강렬하게 읽힘).
**금지:** 상호작용 콘텐츠; 오류 표면으로서의 툴팁; 명확히 라벨된 요소의 툴팁; 반전/대비 뒤집힌 스타일링.

## Toast

일시적 결과 알림, 우측 하단 스택, 최대 3개. `bg.raised-2` 패널, 1px `border.default`, `shadow.overlay`, `fg.primary` `body-sm` 텍스트, 반경 `md`, 선행 상태 아이콘(`status.*` 색상), 선택적 단일 액션(`fg.link` 텍스트 버튼), 5초 자동 소멸(오류: 8초 + 수동 소멸), `slow` 슬라이드+페이드. **실행 취소 관례(v6.0):** 되돌리기 쉬운 경미한 변경(보관, 뷰에서 제거, 소프트 삭제 기반의 단일 삭제)은 사전 확인 대신 8초의 Undo 액션이 있는 Toast로 확인 — 작업이 안전하게 되돌릴 수 있을 때 Popconfirm보다 실행 취소를 선호; 한 액션에 둘이 함께 나타나지 않음. 동일 스킴 표면: 라이트 모드에서 라이트, 다크 모드에서 다크(v2.0에서 변경). **첫 줄 정렬(v6.27.2):** 콘텐츠는 첫 텍스트 줄에 위쪽 정렬 — 아이콘 +2px, 액션 링크는 텍스트 줄 높이에, 소멸 ×는 컴팩트한 20px 박스로(Toast와 인용 바의 인라인 소멸 어포던스는 폼 컨트롤이 아님; 컨트롤 높이 스케일이 적용되지 않음). 후행 컨트롤은 결코 줄바꿈된 블록에 대해 중앙 정렬하지 않음.
**금지:** 검증 오류용 토스트(필드에 인라인); 결정을 요구하는 토스트(Modal); 3개 초과 스택(대신 큐); 반전/대비 뒤집힌 스타일링.

## Banner / Alert

페이지나 섹션을 위한 지속적 인라인 고지. 컨테이너의 전체 너비, 반경 `md`, 패딩 12/16: 16px 아이콘 + `body-sm` 텍스트 + 선택적 액션 링크 + 선택적 소멸.

**색상 변형:** `neutral`(비상태 고지: 예정된 점검 창, 정보성 맥락 — `bg.sunken`, `fg.secondary`), `info`, `success`, `warning`, `danger`.
**강조 변형:**

| Emphasis | Rendering | Use |
|---|---|---|
| `subtle`(기본) | 테두리 없는 `status.*-bg` 채움, `status.*` 텍스트 — 테두리 없음, 레일 없음(v6.4 왼쪽 레일의 v6.7.1 유지관리자 번복: 낡음; 채움 + 색상 텍스트만이 현대적 렌더링. 틴트 + 채도 높은 아웃라인은 여전히 금지된 와이어프레임 공식) | 페이지 및 섹션 수준 고지. |
| `solid` | `status.*-bg-solid` 채움 + `fg.on-solid` 흰색 **semibold(600)** — solid 채움에 결코 regular 무게 아님, 테두리 없음 | 시스템 치명적, 모든 크롬 위에 고정된 앱 전역 스트립(중단, 청구 잠금, 강제 업그레이드). Full-bleed, 반경 없음, 앱 전체에 최대 하나, 항상 액션과 함께. 메시지는 짧게 유지 — solid 라벨 대비 정책(foundations §8)이 적용됨. |

**금지:** 영역당 하나를 초과하는 Banner; 사용자가 그냥 소멸할 수 있는 것에 `solid`; 마케팅 표면으로서의 Banner.

## Avatar

사용자/에이전트 정체성. 크기 20/24/32/40/56px(20 = 밀집 표 셀과 인라인 멘션; 56 = 프로필 표면). 사람은 `full` 반경; 에이전트는 사각형(`sm` 반경) — 필수 제품 언어, 형태만으로 저작자를 스캔해야 함. 이미지, 또는 20% 불투명도의 결정적 `viz` 팔레트 배경에 일치하는 600 무게 텍스트로 이니셜(라틴 2글자 / 한글 1음절).

**상태 표시자:** 선택적 점, 우측 하단, 2px `bg.page` 링. 아바타별 크기: 24→8px, 32→10px, 40→12px, 56→14px; 20px 아바타는 결코 점을 지니지 않음(그 스케일에서 판독 불가 — 상태는 행의 다른 곳에 노출). 사람: 존재(`status.success-bg-solid` = 활성, `border.strong` = 자리 비움). 에이전트: 실행 상태(`status.info-bg-solid` 펄스 = 실행 중, `status.danger-bg-solid` = 실패, 없음 = 유휴). 점은 중간 `-bg-solid` 값 사용, 결코 더 어두운 텍스트 토큰 아님. 제품 표면당 하나의 어휘 — 한 뷰에 두 의미를 결코 함께 두지 않음.
**AvatarGroup:** 겹치는 스택(오프셋 −25%, 각각 2px `bg.page` 링), 최대 4개 보임 + "+N" 오버플로 원(`bg.sunken`, `micro` 텍스트; 클릭 → 전체를 나열하는 popover). 사람과 에이전트가 한 그룹에 섞일 수 있음; 순서는 사람 먼저, 그다음 에이전트.
**금지:** 직사각형 사람 아바타나 둥근 에이전트 아바타; 확립된 어휘 없는 상태 점; 오버플로 카운트를 숨기는 그룹.

## Skeleton · Spinner

Skeleton(선호): `bg.sunken` 블록, 반경 `xs`, 은은한 1.5초 불투명도 맥동, 실제 레이아웃을 반영 — 형태가 알려진 300ms 초과의 모든 로드에. **프리셋 형태(v6.0):** `line`(로컬 타입 스타일 높이의 텍스트 행 하나), `block`(대상 치수의 사각형), `circle`(아바타 크기의 아바타 플레이스홀더) — 이 셋을 조합; 자유 형태 스켈레톤은 금지. Spinner(16/20px, `fg.tertiary` 스트로크): 컨트롤 안과 형태를 모르는 로드에만. **금지:** 레이아웃을 아는데 전체 페이지 스피너; 로드된 레이아웃과 맞지 않는 스켈레톤; 영역당 하나를 초과하는 스피너.

## EmptyState

모든 목록, 표, 검색은 반드시 하나를 가져야 함. 콘텐츠 영역 중앙: 48px `bg.sunken` **메달리온 — 두 개의 동심 헤어라인 링**(−8px의 1px `border.default`, −16px의 1px `border.subtle`; v6.4, 순수 테두리, 그라디언트/글로우 없음) 안의 선택적 24px 아이콘 · 한 줄 제목(16 semibold) · 한 줄 설명(body, `fg.secondary`) · 선택적 단일 액션(첫 객체를 만드는 경우 primary, 그 외에는 secondary). 오류 맛은 메달리온을 `status.danger-bg`/`status.danger`로 틴트.
세 가지 허용된 맛: first-use("Create your first …" — KO: "첫 … 만들기"), no-results("No matches for '{query}'" + 필터 해제 액션), error("Couldn't load …" + 재시도 액션).
**`compact` 변형(v6.0):** 작은 컨테이너용(Drawer 섹션, popover, NotificationCenter, HoverCard 본체): 아이콘 원 없음, 단일 `body-sm` `fg.secondary` 줄 + 선택적 인라인 Link 액션, 패딩 16. 작은 오버레이 안의 전체 크기 EmptyState는 금지 — compact 사용.
**금지:** 일러스트레이션(v1에는 일러스트레이션 언어가 없음); 여러 단락 설명; 액션이 가능한데 액션 없는 막다른 empty state.

## CommandPalette

**목적:** 내비게이션, 액션, 에이전트에게 묻기를 위한 보편적 키보드 우선 진입점. 어디서든 ⌘K / Ctrl+K로 열림; 상단 바 검색 어포던스를 통해서도.

**해부:** 중앙 오버레이, 560px 너비, 상단에서 15vh 오프셋, **유리 재질, scrim 없음**(foundations §5, v6.31: 서리가 포커스 장치 — scrim 위 유리는 탁해짐; esc/클릭 어웨이 소멸은 이전과 같음; 투명도 감소 → 불투명 `bg.raised`), 반경 `lg`, `shadow.modal`. 검색 입력(테두리 없음, 16px, 전체 너비, 선행 검색 아이콘) · 결과 목록(최대 8개 보임, 그 후 스크롤) · 푸터 스트립(`.sy-kbd`를 사용한 11px `fg.tertiary` 키보드 힌트).
**결과:** 11px medium `fg.tertiary` 그룹 라벨 아래 그룹화(Recent / Navigation / Actions / Agents). 행: 높이 40px, 16px 선행 아이콘, 13px 라벨, 후행 `.sy-kbd` 단축키 또는 `fg.tertiary` 맥락; 선택된 행 `bg.selected`. AI를 호출하는 액션은 사각형 에이전트 글리프를 지님.
**상태:** 빈 쿼리 → 최근 항목; 결과 없음 → 단일 EmptyState 스타일 행 더하기 필수 최종 폴백 행 **"Ask agent: '{query}'"**(`ai.fg` 텍스트 + 사각형 글리프) — 팔레트는 결코 막다르지 않음; loading → 3개 스켈레톤 행.
**동작:** 전체 키보드(방향키, Enter, Esc, ⌘K 토글); 타입어헤드가 즉시 필터링(로컬 <50ms, 비동기 결과는 그룹 아래 덧붙음); 실행하면 팔레트가 닫힘; 닫을 때 포커스가 호출 맥락으로 돌아감.
**금지:** 하나를 초과하는 팔레트; 폼 컨테이너로서의 팔레트; 마우스 전용 어포던스; 일치 부분 문자열을 하이라이트하지 않는 퍼지 매치 결과.
**이중 언어:** 일치는 두 로케일의 라벨과 한글의 합리적 로마자화 전반에서 작동해야 함; 그룹 라벨은 로케일화; 결과 행은 결코 라벨을 잘라내지 않음(대신 후행 맥락을 잘라냄).

## ProgressBar

**목적:** 오래 실행되는 작업의 진행률(에이전트 실행, 가져오기, 배치 작업). 언제 필수인지는 `ai-patterns.md` §11 참조.

**해부:** 4px 트랙(`bg.sunken`, 반경 `full`) + 채움(반경 `full`). 위의 선택적 라벨 행: 13px 설명 좌측, "N of M" 또는 퍼센트 우측(`fg.tertiary`, tabular-nums).
**변형:** `default`(채움 `meter.fill` — 중립 중간 회색, 사용자가 시작한 작업; v3.6에서 키-검정에서 변경), `ai`(채움 `ai.solid` — 에이전트 실행; v6.19, accent-bg였음). **확정** 채움은 `base` 지속으로 너비 애니메이션; **미확정**은 1.2초 간격으로 30% 너비 세그먼트가 휩쓸림을 보임.
**상태:** in-progress, success(채움이 한 `slow` 박자 동안 `status.success-bg-solid`로 전환된 후 바가 완료 상태로 대체됨), failed(채움 `status.danger-bg-solid`, 아래 오류 텍스트와 함께 바 지속), cancelled(채움 `border.strong`).
**사용량 미터 관할(v5.2):** 플랜/토큰/할당량 소비는 ProgressBar의 허용된 "용량" 용도 — 확정, `meter.fill`, ≥80%에서 `status.warning-bg-solid`로 그리고 100%에서 `status.danger-bg-solid`로 전환(플랜 한도 Banner 에스컬레이션과 함께). 설정/청구 표면에 위치하며 한도가 가까울 때 Composer에 인접한 컴팩트 바로 나타날 수 있음; 결코 영구적 내비게이션 크롬으로는 아님. 값은 tabular-nums("1.2M / 2M tokens").
**금지:** 4px 외의 높이; 알려진 분모 없는 퍼센트 텍스트; 정적 "score" 표시(대신 stat Card 조합 — 할당량 소비가 유일하게 허용된 용량 미터); 영역당 하나를 초과하는 미확정 바.

## Composer

**목적:** Console와 인라인 ask-agent 표면을 위한 메시지/지시 입력 — AgentOS에서 가장 많이 쓰이는 단일 컨트롤. (v5.2 추가.)

**해부:** 컨테이너(`bg.sunken` 채움, 반경 `lg`, 컨테이너의 포커스 링) · 선택적 ComposerQuote 바(ai-patterns §18: ai.surface, 반경 `xs`, 한 줄 + ×, 최대 하나) · 텍스트 영역 위의 첨부 행(**종류별로 그룹화, v6.28.1: 이미지 타일은 한 행에, 문서 칩은 그 아래 행에 — 종류 안에서는 시간순, 비면 행이 사라짐; 첨부는 §28 안내 캡션을 지닐 수 있음; §27의 zero-state 스타터가 트레이 위에 렌더; §30의 ghost 텍스트 연속.

**전송 ↔ 중지 변형:** 에이전트가 생성하는 동안 전송 버튼은 같은 위치에서 **중지**(사각형 아이콘, `secondary` 토널)가 됨 — 결코 두 번째 버튼이 아니고, 결코 비활성화되지 않음. Composer 입력 자체는 생성 중 결코 비활성화되지 않음(ai-patterns §2): 타이핑은 계속됨; Enter는 제품 설정에 따라 큐에 넣거나 중단함.
**슬래시 커맨드(v6.15):** 빈 Composer 시작의 `/`는 커맨드 팔레트를 에이전트 액션으로 범위 지정; 타이핑하는 동안 단일 최선 일치의 나머지가 `fg.placeholder` ghost 텍스트로 렌더되어 →로 수락(결코 Tab 아님 — IME 충돌), 계속 타이핑하면 해제됨. 커맨드는 닫힌 액션 글로서리에서만 옴; 자유 텍스트는 결코 완성을 트리거하지 않음.
**키보드:** Enter 전송, Shift+Enter 줄바꿈 — 그리고 **IME 조합 중 Enter는 결코 전송하지 않음**(한국어/일본어 입력은 Enter로 조합함; 조합 중 전송은 전형적인 KO 입력 버그). Kbd 힌트(`micro`, `fg.tertiary`)는 포커스 시 아래, 첫 실행만.
**상태:** default, focus(컨테이너의 링), generating(중지 변형), send-failed(위에 인라인 `caption` `status.danger` 오류 + 재시도 텍스트 버튼; 초안 보존), empty(전송 비활성화가 유일하게 허용된 비활성화 — 보낼 것이 없음).
**초안:** 콘텐츠는 대화별로 내비게이션 전반에 지속; 결코 조용히 폐기되지 않음.
**금지:** 생성 중 composer 비활성화; 툴바 잡동사니(서식 버튼 — 에이전트 입력은 평문 + 첨부); 화면당 하나를 초과하는 Composer; 포커스 시 사라지는 지시를 담은 플레이스홀더(kbd 힌트 사용).

## ResponseToolbar

**목적:** 에이전트 메시지에 대한 액션: 복사 · 재생성 · 피드백(엄지) · 오버플로. (v5.2 추가.)

**해부:** `ghost` 아이콘 버튼(16px 아이콘)의 행, 에이전트 메시지의 좌측 하단, 평상시 `fg.tertiary`. 고정 순서: 복사, 재생성, 엄지 위, 엄지 아래, ⋯ 오버플로.
**`media` 변형(v6.16, 관할 v6.17.7에서 강화):** MediaGroup 옆에 앵커된 세로 pill 레일(raised 채움, `border.overlay`, `shadow.raised`, 반경 `full`, 4px 패딩) — 복사 + 엄지만, 호버 노출, 그룹당 레일 하나. MediaGroup이 메시지의 유일한 콘텐츠일 때만: 미디어가 텍스트를 동반하면 메시지 수준 툴바가 답변 전체를 관장하고 레일은 금지됨(한 메시지에 두 피드백 표면은 중복).
**배치:** 데스크톱 focus 표면에서 호버/포커스 노출; 밀집 콘솔에서 지속적(그곳에서는 호버 의존 어포던스에 지속적 폴백이 필요).
**동작:** 복사는 마크다운 소스를 복사(Toast 확인); 재생성은 최신 에이전트 메시지에만(이전 메시지는 그것을 잃음); 엄지 선택 상태 = 버튼의 표준 반경에서 `bg.selected` 채움 + `fg.primary` 스트로크(v6.47.1 — `border.focus` 파랑 + 원이었음, 이후 두 법칙을 위반하는 v5.2 유물: 파랑은 포커스/링크/status.info용으로 예약되고, 선택은 형태가 아니라 채움 + 잉크로 읽힘; Composer 전송 버튼이 유일한 원형 컨트롤로 남음)(결코 채워진 아이콘 아님 — 스트로크 집합만; 즐겨찾기 별이 유일한 활성-시-채움 예외), 상호 배타적, 탭하여 실행 취소. **광학적 중앙 정렬(v6.47.2, 숫자 미세 조정 패밀리):** 엄지 글리프는 비대칭 잉크를 지님(위는 오른쪽 무거움, 아래는 왼쪽 무거움 — 16px 렌더에서 기하학적 중심으로부터 ~1.5px 벗어남); 둘 다 잉크가 보이는 채움 안에서 중앙에 오도록 ∓1px translateX를 얻음. 엄지 쌍이 렌더되는 어디서나 적용(메시지 툴바 + 미디어 레일); 엄지 아래는 한 필드 코멘트 Popover("What went wrong?" / "어떤 점이 아쉬웠나요?")를 열 수 있으나, 결코 필수는 아님.
**관할:** 에이전트 메시지만 — 결코 사람 말풍선에, 결코 ProposalCard에(그것들은 자체 푸터를 가짐).
**금지:** 툴바 안 파괴적 액션; ⋯ 오버플로 밖의 공유/내보내기 액션; 여기 외 어디서든 피드백 아이콘.

## ProposalCard

**목적:** 중대한 에이전트 액션의 human-in-the-loop 승인. 전체 동작: `ai-patterns.md` §5.

**이중 언어:** 결과 진술은 두 로케일 모두에서 카운트 + 명사를 명시; 버튼 라벨은 결코 잘리지 않음.

## DescriptionList

## ButtonGroup

**목적:** 관련 액션의 시각적으로 붙은 그룹. (v2.0 갭 감사 추가.)

**변형:**

| Variant | Composition | Use |
|---|---|---|
| `attached` | 하나의 형태로 융합된 2–4개 `secondary`(토널) 버튼(반경은 바깥 모서리에만; 세그먼트 사이 1px `border.default` 구분자) | 함께 조작되는 동급 액션: 확대/축소, 이전/다음, 액션이 있는 뷰 토글. |
| `split` | 주요 액션 버튼 + 대안 액션의 Menu를 여는 붙은 셰브런 버튼 | "Run" + ▾ (Run with options…, Schedule run…). 메뉴의 액션은 주요 액션의 진정한 변형이어야 함. |

split은 주요 버튼의 변형을 따름(`primary` 또는 `secondary`; `brand`는 포인트 컬러 CTA이며 split 버튼에는 쓰지 않음); 셰브런 세그먼트는 너비 28px, 동일 높이/변형, 테두리 색상의 1px 구분선으로 구분(또는 solid 채움에서는 `alpha.white-16`).
**금지:** `primary` 버튼끼리 붙이기(영역당 primary 하나 원칙은 여전히 유효); 메뉴가 무관한 액션을 담은 split 버튼; 4개 초과 세그먼트; SegmentedControl(배타적 상태)이 실제 필요일 때 `attached` 사용.
**접근성:** 그룹 라벨이 있는 `role="group"`; split의 셰브런은 `aria-haspopup` + 자체 라벨을 얻음("More run options").

## SourceChip

**목적:** 에이전트 주장에 대한 인라인 출처 표식. 전체 동작: `ai-patterns.md` §6. **표식 해부(v6.37 — 유지관리자 기준: 원형 위첨자 스타일):** 지지되는 문장 끝의 18px 둥근 칩(`radius.full`, **중립 `bg.sunken` 채움, `fg.secondary` 맨 숫자** — 괄호 없음, 11px semibold tabular, mono가 아니라 sans, **1px 위쪽 패딩 광학 미세 조정 — Badge md 미세 조정 선례: Pretendard의 11px 숫자는 작은 컨테이너에서 line-height 1일 때 높이 앉음; pill의 숫자 원에도 적용, v6.38.4**), `text-bottom` 정렬; 호버/링크-하이라이트 시 `ai.surface-hover`로 단계 이동. 괄호 mono `[n]` 형태는 은퇴함.

**해부:** 18px 높이, `bg.sunken`, 반경 `xs`, mono 11 숫자(`[1]`), 지지되는 문장 뒤에 배치, 2px 베이스라인 오프셋. 호버/클릭 → 소스 제목, 출처 아이콘, 타임스탬프, 링크 열기가 있는 Popover. 메시지 푸터는 모든 소스를 나열(13px `fg.secondary`).
**상태:** default, hover(`bg.selected`), broken(소스를 더 이상 가져올 수 없음 — 칩이 `status.warning` 숫자를 얻고 popover가 그렇다고 말함).
**금지:** 장식적 사용, 자기 인용, 가짜 번호 매기기, 문장당 3개 초과 칩(대신 종합); 이것 외의 각주 스타일.

---
**출처 행(v6.36, 레이아웃 v6.37.1):** 평문 각주 줄은 은퇴함 — `micro-label` "출처" 아이브로가 **pill 형태** 소스 카드 행 위에 자체 줄로 앉음 — 테두리 없는 `bg.sunken` 채움; 해부(v6.38.2): **선행 18px 페이지 채움 숫자 원**(11/1 semibold tabular — 인라인 표식을 구조적으로 반영: 둘 다 텍스트 스케일과 pill 스케일의 번호 매긴 원) + **12px 소스 유형 아이콘(`fg.tertiary`)** + 소스 이름(12/18, `fg.secondary`), 간격 6, 원이 고르게 앉도록 2px 선행 인셋과 함께 패딩 2×8(아이콘 v6.38.3에서 복원, 유지관리자 선호 — 숫자가 원에 담기면서 아이콘이 잡동사니가 아니라 깔끔한 메타데이터로 읽힘; 여러 소스가 한 행을 공유할 때 스캔을 도움). 회색 사다리의 다음 단계에서 호버. pill 실루엣은 Console을 공유하는 rounded suggestion Chip과 의도적으로 다름 — 두 채팅 칩 종은 형태만으로 구별 가능해야 함; 소스 pill은 ContextCard 파생물로, Chip 패밀리의 rounded 규칙 밖. 카드를 호버하면 그 인라인 [n] 표식이 하이라이트되고 그 역도 같음 — 번호 매핑이 정신적이 아니라 시각적이 됨. **히트 대상(v6.36):** 18px 칩은 시각적 크기를 유지하되 보이지 않는 ≥24px 히트 영역을 지님(인셋 −4 pseudo) — 24px 하한은 글리프가 아니라 대상에 적용됨.

## ContextCard

**목적:** Console 스레드와 Composer에서 물리적 카드로 렌더된 참조 객체(문서, 회의, 표, 페이지) — "에이전트가 보고 있는 것"의 보이는 형태. (v6.14 추가.)

**해부:** 아웃라인 `bg.raised` 카드, 반경 `sm`, 패딩 8×12, 최대 너비 240: 24px `bg.sunken` 타일(반경 `xs`, 동심 예외: 흐르는 콘텐츠) 위의 객체 유형 아이콘(14px) · 제목(13 medium, 한 줄, 말줄임) · `caption` 메타(소스 · 날짜), 최대 한 줄. 호버: `border.strong`; 클릭하면 객체 열림.
**스택:** 참조 객체 2개 이상은 평평하게 쌓임 — −4px의 단일 오프셋 하단 카드와 2px 페이지 색상 아웃라인 링으로 분리된 상단 카드(카운트 오버레이 선례), 더하기 상단 카드의 `+N` Badge. 보이는 객체 최대 3개; 펼치면 세로로 나열. **회전 없음** — 장난스러움이 아니라 정밀함.
**컴팩트(Composer @-멘션):** 아이콘 타일 16px + 제목만, 패딩 2×8; Composer 텍스트 행에 인라인으로 렌더.
**금지:** 카드 안 썸네일이나 콘텐츠 미리보기(v1에는 미리보기 언어가 없음); 하나를 초과하는 메타 줄; 수동 색상; 참조 객체 외의 것에 스택 사용.

## DiffView

**목적:** 독립형 변경 비교 — 실행 설정, 정책 편집; ProposalCard diff 규칙의 승격. (P3.)

**해부:** 기본은 통합 레이아웃; 설정 비교를 위해 ≥960px에서 나란히 변형 허용. 줄 행: 거터(+/− 표식 + 선택적 줄 번호) + mono 12 콘텐츠. 추가 = `status.success-bg`, 제거 = `status.danger-bg`, 단어 수준 변경은 줄 안에서 더 어두운 틴트 — 결코 색상 단독이 아님(거터 표식이 색맹 안전 채널). 변경 없는 실행은 펼침 가능한 "⋯ 24 unchanged lines" 행으로 접힘. 헤더: 객체 이름 + 변경 카운트(+12 −4, tabular).
**금지:** 삼방향 diff(v1 범위 밖); diff 안 구문 강조(한 번에 하나의 신호); 비동기화될 수 있는 스크롤 연동 창.

## MediaGroup

**목적:** 에이전트가 **생성한** 미디어(이미지, 차트 렌더, 미리보기)를 Console 답변에서 캐주얼한 부채꼴로 제시 — 시스템의 유일하게 허용된 장난스러운 순간. (v6.16 추가.)

**해부:** 카드 `bg.page`, 1px `border.default`, 반경 `md`, 분리를 위한 2px 페이지 색상 아웃라인 링, 오버플로 숨김; 위에 object-fit cover의 미디어 영역 + 아래 **캡션 스트립**(`bg.surface`, 헤어라인 위쪽 규칙, `micro` 라벨; `+N` Badge는 스트립에 우측 정렬 — v6.17.6). 카드 2–3개는 **±2.5° 교대 회전**과 ~20% 겹침으로 부채꼴; 단일 항목은 평평하게 렌더. 보이는 최대 3개 + 마지막 카드의 `+N` Badge; 클릭하면 미디어 뷰어 열림.
**동작:** 호버/포커스는 카드를 곧게 펴고(회전 → 0) `fast`로 들어 올림(`shadow.raised`); `prefers-reduced-motion`은 그룹 전체를 평평하고 정적으로 렌더.
**관할(강함):** 에이전트 답변 안의 에이전트 생성 미디어만. 참조/첨부 객체는 ContextCard 사용(항상 평평); 다른 곳의 썸네일은 금지. **회전은 시스템 어디에도 존재하지 않음** — design.md §8 참조.
**원칙(v6.16):** 장난스러움은 에이전트의 *출력*에 살지, 결코 크롬에 살지 않음.

## Slider · NumberInput

**목적:** 경계 지어진 연속 값(Slider)과 정밀한 숫자 입력(NumberInput). 그들은 짝을 이룸; 어느 것도 다른 것을 대체하지 않음. (P3.)

**Slider:** 4px 트랙(`bg.sunken`) + `action.primary-bg` 채움 + 16px 썸(포커스 링); 드래그하는 동안 썸 위에 값 라벨이 나타남(Tooltip 표면); 열거된 정지점의 선택적 눈금 표시; 방향키 단계, Shift = 10×; `range` 변형 = 두 썸, `bg.selected`… 아니 — 썸 사이 채움은 표준 채움 사용. 관할: *위치가 의미인* 값(임계값, 샘플링 %, 볼륨). 결코 보이는 숫자 값 없이 안 됨; 결코 무경계나 정밀도 중요 값에 안 됨.
**NumberInput:** 채움형 Input + 쌓인 스테퍼(셰브런, 각 24px 히트) + 선택적 단위 접미사(`fg.tertiary`); 타이핑 입력 자유, blur 시 `caption` 노트와 함께 min/max로 클램프; 방향키 단계. Tabular 숫자.
**금지:** 날짜/시간용 슬라이더; 비숫자 입력의 스테퍼; 분모를 모르는 퍼센트 슬라이더.

## ChoiceCard

**목적:** 옵션이 설명을 받을 만한 일대일 또는 다대일 선택 — 플랜 선택기, 에이전트 유형 선택, 온보딩 결정. (P3.)

**해부:** `outlined` Card의 그리드(2–3열, 동일 높이), 패딩 16: 선택적 20px 레지스트리 아이콘 · 제목(`heading-sm`) · 설명(`caption` `fg.secondary`, ≤2줄). 선택 = 1px `border.selected` 보더 + 그 바깥에 밀착한 4px 더 옅은 `border.subtle` 아웃라인(오프셋 0 — 간격 없이, 얇은 선택 선과 더 넓은 그레이 밴드가 두꺼운 단일 보더가 아니라 하나의 연속된 투톤 헤일로로 읽힘) + 우상단 16px 체크. 기본은 Radio 시맨틱; `multi` 변형은 체크박스 시맨틱을 쓰고 Checkbox를 보임.
**관할:** 설명이 필요한 2–6개 옵션. 6개 초과나 맨 라벨 → Radio/Select.
**접근성:** `radiogroup`/`group`, 방향키 내비게이션, 카드 전체가 대상.
**금지:** 선택 카드 안 입력 중첩; 한 그룹에서 불균등한 카드 높이; 이미지/일러스트레이션(v1에는 없음).

## HoverCard

**목적:** 호버/포커스 시 엔티티 참조(사용자, 에이전트, 실행)의 풍부한 미리보기. (P3.)

**해부:** popover ≤320px, `bg.raised-2`, `border.overlay`, `shadow.overlay`, `z.dropdown`: Avatar 헤더(이름 + `caption` 역할/상태) + 2–4개 DescriptionList 행 + 선택적 단일 ghost 액션. 500ms 호버 후 또는 포커스 시 열림; 떠남/Esc 시 닫힘.
**규칙:** 호버는 향상이지 결코 요구사항이 아님 — HoverCard의 모든 것은 엔티티의 클릭스루 페이지에도 존재해야 함(키보드와 보조 접근이 호버에 의존해서는 안 됨).
**금지:** 폼; 중첩 HoverCard; 이미 popover를 여는 요소의 호버 카드.

## Popconfirm

**목적:** 낮은 위험의 되돌리기 가능한 듯한 액션을 위한 인라인 확인 — 확인 없음과 Modal 사이의 단계. (P3.)

**해부:** 트리거에 앵커된 popover(`z.dropdown`): 결과를 명시하는 질문 하나(`body-sm`) + Cancel(`ghost` sm) + 확인(`danger`/`primary` sm). Esc/바깥 클릭으로 취소.
**관할:** 결과를 쉽게 재생성할 수 있는 단일 객체 액션(고정 해제, 저장된 필터 지우기, 첨부 하나 제거). 영구적 데이터 손실, 대량 작업, 또는 명시된 카운트가 있는 것 → Modal(R5 규칙).
**금지:** popconfirm 연쇄; 안에 입력; Modal이 마땅한 액션을 부드럽게 하는 데 사용.

## ContextMenu

**목적:** 밀집 데이터 표면의 우클릭 메뉴. (P3.)

**해부 및 동작:** Menu 컴포넌트, 포인터 위치에서 열림(`z.dropdown`); 동일한 항목 규칙(레지스트리 아이콘, 파괴적 항목 마지막, full-bleed 구분선, 4px 간격).
**중복 규칙:** 모든 컨텍스트 메뉴 액션은 보이는 UI에도 존재해야 함(행 액션, 툴바, ⋯ 메뉴) — 컨텍스트 메뉴는 가속기이지, 결코 유일한 경로가 아님(발견 가능성 — 포인터로 숨겨진 어포던스가 어떤 액션도 막아서는 안 됨).
**관할:** 밀집 표, 트리, 캔버스 같은 표면. Focus 아키타입 산문에는 아님.
**금지:** 컨텍스트 전용 액션; 한 수준을 넘는 서브메뉴; 텍스트 선택에서 브라우저 컨텍스트 메뉴 재정의.

## CalendarView

**목적:** 예약 시각화 — 에이전트가 언제 실행되는지, 무엇이 실행되었는지. 예약 시스템이 아님. (P3.)

**월 그리드:** 확대된 DatePicker 셀 해부 — 날짜 셀 ≥96px 높이, 날짜 번호 `label-sm`(오늘 = 아웃라인 pill), 이벤트는 점 + `body-sm` 텍스트 행(이벤트 점은 카테고리 색상 규칙을 따름: 시스템 할당 viz 틴트), 셀당 최대 3개 그 후 "+N" → 전체를 나열하는 popover.
**주 변형:** 이벤트 블록이 있는 시간 행(48px); 블록 = viz 틴트 채움 + `label-sm`; 겹침은 너비를 분할.
**동작:** 이벤트 클릭 → DescriptionList + 열기 액션이 있는 popover; 드래그로 재예약은 예약 모델이 허용하는 곳에서만, Toast 확인 + 실행 취소와 함께.
**금지:** 내부적으로 스크롤하는 월 셀; 빈 날 플레이스홀더(빈 날은 그냥 비어 있음); 하나를 초과하는 이벤트 색상 시스템(viz 할당만).

## NotificationCenter

**목적:** 벨의 패널 — 자리를 비운 동안 무슨 일이 있었는지. (P3.)

**해부:** popover 패널 360px(`z.dropdown`): 헤더("Notifications" / "알림" + 설정과 모두 읽음 표시 ghost 아이콘 버튼) · **필터 SegmentedControl**(전체 / 읽지 않음 / 멘션, 전체 너비, 즉시 필터) · Today / Earlier로 그룹화된 스크롤 목록(`label-sm`; 필터로 항목이 비면 그룹 헤더는 숨김) · 항목: **선행 마커** — 액터 이벤트는 액터 Avatar 20(원형 사람 / 사각형 에이전트), 시스템 이벤트는 중립 20px 타입 메달리온 — 에 템플릿 문장(content.md 동사), `caption` 타임스탬프, 선택적 단일 인라인 액션; 미읽음 = `bg.surface` 채움 + 왼쪽 가장자리의 8px `status.info-bg-solid` 점 · 푸터: 전체 페이지로 가는 단일 전체 너비 "View all" ghost 버튼(30개 항목 이후).
**항목 유형(v6.59):** `run`(에이전트 실행 완료), `approval`(검토 대기 제안 — 그 단일 인라인 액션은 ProposalCard를 *열기만* 함), `mention`(멘션 필터가 고르는 것), `comment`, `system`. 유형은 선행 마커와 동사로 전달되며 — 결코 틴트만으로는 아님.
**항목별 컨트롤(v6.59):** 행에 호버하거나 포커스하면 행의 오른쪽 가장자리 위로 작은 컨트롤 묶음이 떠오름 — 읽음/안읽음 전환과 지우기(×) — 타임스탬프를 가리거나 행 높이를 바꾸지 않음(호버 표시는 오버레이를 *더할* 뿐, 기존 콘텐츠를 제거하거나 행을 재배치하지 않음); 헤더의 모두 읽음 표시는 모든 미읽음을 한 번에 지움. 이들은 크롬이며, 단일 콘텐츠 인라인 액션과 별개임.
**동작:** 항목 클릭은 객체로 이동하고 읽음 표시. 인라인 액션이 있으면 중대한 행위의 표면을 *열기만* 함 — 에이전트 제안 승인은 ProposalCard에서 일어나지, 결코 알림에서 아님. 읽지 않음/멘션 필터와 읽음/지우기 컨트롤은 결코 숫자를 표시하지 않음 — 벨의 카운트 오버레이가 유일한 카운트.
**Empty state:** 컴팩트 맛("You're all caught up" / "모두 확인했습니다"), 필터가 아무것도 내지 않거나 목록이 비워졌을 때 표시.
**금지:** 마케팅 콘텐츠; 항목당 하나를 초과하는 콘텐츠 인라인 액션(읽음/지우기 호버 컨트롤은 예외 — 콘텐츠 액션이 아니라 크롬); 벨의 카운트 오버레이 외의 미읽음 카운트(99+ 규칙); 색/틴트만으로 표현한 유형.

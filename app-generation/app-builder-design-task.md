# App Builder — 디자이너 할 일 목록

## 크게 2개를 디자인해야 함

### A. 빌더 화면 (앱을 만드는 사람이 보는 화면)
### B. 완성된 앱 화면 (만들어진 앱을 쓰는 사람이 보는 화면)

---

## A. 빌더 화면

사용자가 "대시보드 만들어줘"라고 했을 때 거치는 과정.

### A-1. 채팅 패널
- [ ] AI 메시지 말풍선
- [ ] 사용자 메시지 말풍선
- [ ] 프롬프트 입력창 + 전송 버튼
- [ ] 추천 프롬프트 칩
- [ ] 파일 첨부 버튼 (프롬프트에서 데이터 올리기)
- [ ] "생성 중..." 로딩 애니메이션

### A-2. 데이터 소스 연결
- [ ] 파일 업로드 영역 (드래그앤드롭)
- [ ] 업로드된 파일 목록 (파일명, 크기, 상태 배지)
- [ ] 감지된 스키마 미리보기 (컬럼명, 행 수)
- [ ] DB 직접 연결 화면 (나중에, 지금은 파일만)

### A-3. 레이아웃 편집
- [ ] 위젯 팔레트 (드래그 가능한 컴포넌트 목록)
- [ ] 캔버스 (그리드, 위젯 배치 영역)
- [ ] 위젯 카드 (드래그 핸들, 리사이즈 핸들)
- [ ] 탭 바 (페이지 추가/삭제/이름 변경)
- [ ] 위젯 선택 시 우측 설정 패널 (데이터 바인딩, 스타일)

### A-4. 앱 설정
- [ ] 토글 스위치 목록 (실시간 갱신, 반응형, 내보내기 등)
- [ ] 테마 선택 (다크/라이트/커스텀 컬러)
- [ ] 배포 버튼 + 확인 모달

### A-5. 빌드 진행 상태
- [ ] 단계별 체크리스트 (완료/진행중/대기)
- [ ] 프로그레스 바 or 스텝 인디케이터

---

## B. 완성된 앱 화면

만들어진 앱을 사용자가 실제로 쓸 때 보는 화면.
각 컴포넌트의 **상태별** 디자인이 필요함.

### B-1. 글로벌 필터 바
- [ ] 필터 바 레이아웃 (가로 배치)
- [ ] 드롭다운 (닫힌 상태 / 열린 상태)
- [ ] 다중 선택 칩 (선택된 항목 + X 삭제)
- [ ] 날짜 범위 피커 (캘린더 열린 상태)
- [ ] 슬라이더 (트랙 + 썸)
- [ ] 토글 스위치 (ON / OFF)
- [ ] 검색 입력 (자동완성 드롭다운 열린 상태)
- [ ] 필터 프리셋 칩 (저장된 필터 목록)
- [ ] 필터 저장 모달

### B-2. KPI 카드
- [ ] 기본 상태 (라벨, 숫자, 증감률, 스파크라인)
- [ ] 증가 상태 (초록 화살표)
- [ ] 감소 상태 (빨간 화살표)
- [ ] 로딩 상태 (스켈레톤)
- [ ] 알림 상태 (임계값 초과 시 빨간 테두리)

### B-3. 차트 공통
모든 차트에 필요한 공통 디자인:
- [ ] 차트 카드 프레임 (제목, 옵션 메뉴)
- [ ] 툴팁 (호버 시 데이터 표시)
- [ ] 범례 (위치, 스타일)
- [ ] 빈 상태 ("데이터 없음")
- [ ] 로딩 상태 (스켈레톤)
- [ ] 에러 상태 ("불러오기 실패" + 재시도)
- [ ] 드릴다운 힌트 ("클릭하면 상세 보기")

### B-4. 차트 개별 (ECharts가 그리므로 컬러/레이아웃만)
- [ ] 막대그래프 컬러 규칙 (1색 / 다색 / 조건부)
- [ ] 도넛 내부 텍스트 (합계? 퍼센트?)
- [ ] 게이지 색상 구간 (정상/경고/위험)
- [ ] 히트맵 색상 그라디언트 (연한 파랑 → 진한 파랑)
- [ ] 트리맵 계층별 색상 규칙
- [ ] 워터폴 증가/감소 색상
- [ ] 퍼널 단계별 색상

### B-5. 테이블
- [ ] 기본 테이블 (헤더, 행, 줄무늬)
- [ ] 정렬 아이콘 상태 (미정렬 ↕ / 오름 ▲ / 내림 ▼)
- [ ] 필터 드롭다운 (컬럼 헤더에서 열리는)
- [ ] 검색바
- [ ] 페이지네이션 바
- [ ] 행 호버 상태
- [ ] 행 선택 상태 (체크박스 + 하이라이트)
- [ ] 일괄 작업 바 ("3건 선택됨" + 액션 버튼)
- [ ] 인라인 편집 상태 (셀 클릭 → 입력 모드)
- [ ] 인라인 편집 저장/취소 버튼
- [ ] 조건부 서식 (셀 배경색, 배지, 아이콘)
- [ ] 컬럼 고정 구분선 (그림자)
- [ ] 컬럼 표시/숨김 드롭다운
- [ ] 행 클릭 → 상세 패널 (사이드 슬라이드 or 모달)
- [ ] 빈 상태 ("검색 결과 없음")
- [ ] 로딩 상태 (행 스켈레톤)

### B-6. 피벗 테이블
- [ ] 축 선택 드롭다운 (행/열/값)
- [ ] 합계 행/열 스타일 (볼드, 배경색)

### B-7. 데이터 입력 폼
- [ ] 각 입력 필드 상태:
  - 기본
  - 포커스 (파란 테두리)
  - 에러 (빨간 테두리 + 에러 메시지)
  - 비활성 (회색)
- [ ] 필드 타입별:
  - 텍스트 (단일행)
  - 텍스트 (다중행/textarea)
  - 숫자
  - 드롭다운 (DB 참조 옵션)
  - 날짜 피커
  - 라디오 버튼
  - 체크박스
- [ ] 폼 레이아웃 (2열 그리드)
- [ ] 필수 표시 (빨간 *)
- [ ] 유효성 에러 메시지
- [ ] 중복 체크 결과 (통과/중복 있음)
- [ ] 제출 전 확인 모달 (입력값 미리보기)
- [ ] 성공 토스트 ("저장되었습니다")
- [ ] 입력 이력 로그 리스트

### B-8. 파일 관련
- [ ] 파일 업로드 영역 (드래그앤드롭, 호버 상태)
- [ ] 업로드 진행률 바
- [ ] 파일 목록 (썸네일, 파일명, 크기, 연결된 레코드)
- [ ] 이미지 썸네일 미리보기
- [ ] 파일 액션 (다운로드, 연결, 삭제)

### B-9. 내보내기
- [ ] 내보내기 버튼 (테이블 상단 or 헤더)
- [ ] 포맷 선택 카드 (CSV, Excel, PNG, PDF)
- [ ] "현재 필터 기준으로 내보내기" 안내 문구

### B-10. 알림 및 상태
- [ ] 알림 카드 3종 (critical 빨강 / warning 노랑 / info 파랑)
- [ ] 실시간 갱신 표시 (Live 뱃지 + 마지막 갱신 시각)
- [ ] 스켈레톤 로딩 (카드형 / 테이블형 / 차트형)
- [ ] 에러 상태 (아이콘 + 메시지 + 재시도 버튼)
- [ ] 조건부 색상 변경 예시 (정상→초록, 위험→빨강)

### B-11. 지도
- [ ] 지도 프레임 (모서리 둥글기, 테두리)
- [ ] 마커 스타일 (색상별 의미, 크기)
- [ ] 마커 호버 툴팁
- [ ] 경로 선 스타일 (점선, 색상)
- [ ] 지오펜스 영역 (반투명 테두리)
- [ ] 범례

### B-12. 시뮬레이션
- [ ] 파라미터 슬라이더 (라벨 + 현재값)
- [ ] 결과 요약 (현재 vs 예측, 변동률)
- [ ] 비교 차트 (기존 점선 + 시뮬레이션 실선)

## 디자인 순서 추천

### 1단계: 디자인 토큰 (하루)
컬러, 타이포, 간격, 모서리, 그림자 정의.
이거 없으면 나머지 전부 흔들림.

### 2단계: 공통 컴포넌트 (2-3일)
버튼, 입력 필드, 드롭다운, 배지, 카드, 모달, 토스트.
이것들이 모든 화면에서 재사용됨.

### 3단계: 빌더 화면 (2-3일)
A-1 ~ A-5. 채팅 → 데이터 → 레이아웃 → 설정 → 빌드 순서.

### 4단계: 완성된 앱 화면 (3-5일)
가장 양이 많음. 근데 공통 컴포넌트가 있으면 조합이라 빠름.
KPI → 테이블 → 필터 → 폼 → 차트(컬러만) 순서 추천.

### 5단계: 상태 디자인 (1-2일)
로딩, 에러, 빈 상태, 호버, 포커스, 비활성.
이거 빠지면 개발자가 알아서 만들어서 디자인 깨짐.

## 참고: 안 해도 되는 것
- 차트 내부 디자인: ECharts가 그림. 컬러 팔레트만 정하면 됨.
- 지도 타일: Mapbox/Leaflet이 그림. 마커/범례만 디자인.
- 반응형 브레이크포인트: 일단 데스크탑만. 태블릿/모바일은 v2.
